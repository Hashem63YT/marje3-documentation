مرجع (Marje3)
نظرة عامة الميزات البنية التقنية السب دومينز الخلاصة

مشروع مرجع

الفكرة الأساسية

منصة شاملة وموثوقة لأهل السنة والجماعة، تجمع بين المحتوى العلمي والديني الذي تتم مراجعته من قِبَل علماء ومشايخ أهل السنة والجماعة في العالم الإسلامي.

التفاصيل العامة

تسجيل الدخول والتحقق

  • تسجيل الدخول بحساب Google فقط (لا كلمات سر)
  • OTP للتحقق: مرة واحدة لكل جهاز عند أول تسجيل - إعادة تحقق فقط عند: تسجيل دخول من جهاز جديد، عملية حساسة (تغيير إعدادات أمان)
  • طريقة لتجنب الحسابات الوهمية
  • استعادة الحساب: في حالة فقدان الحساب، يمكن للمستخدم إنشاء حساب جديد أو التواصل مع Google مباشرة (نحن لسنا مسؤولين عن حسابات Google)
  • إدارة الجلسات: مدة الجلسة 30 يوم، Inactivity timeout 24 ساعة، Cookies: httpOnly و secure، Token rotation مفعّل، منع session fixation
  • حد الجلسات المتزامنة: تحديد حد أقصى للجلسات المتزامنة لكل مستخدم

البحث والذكاء الاصطناعي

  • البحث بالذكاء الاصطناعي لتجنب تكرار الأسئلة
  • نظام ذكي للإجابة على الأسئلة الشائعة
  • توصيات محتوى ديناميكية للمستخدم حسب نشاطه السابق
  • Search Index Governance (إدارة فهارس البحث): تحديث فهارس البحث بشكل مجدول، إعادة بناء الفهارس تلقائيًا عند حدوث تغييرات كبيرة في المحتوى، ضمان دقة وحداثة نتائج البحث، Rate-limited Search Index & Content Rebuild لتجنب إساءة الاستخدام

الدعم الفني

  • المحادثة المباشرة للدعم الفني

الإشعارات

  • نظام إشعارات كامل يدعم نصوص، صور، وروابط
  • إمكانية إرسال إشعارات يدوية عبر admin.marje3.app (عنوان + نص + صورة + الأولوية)
  • تخصيص لون الإشعار وشكله
  • إشعارات مؤقتة يمكن حذفها تلقائيًا بعد فترة محددة
  • تحديد مستويات الأولوية لكل إشعار (Top, Medium, Low، ويمكن إضافة مستويات أعلى)
  • إشعارات الطوارئ تظهر لجميع المستخدمين فورًا (مثل الزلازل وتعليمات السلامة)
  • إشعارات مرتبطة بالأنشطة اليومية والموسمية لكل مستخدم
  • التحقق من الروابط: التحقق من صحة الروابط في الإشعارات لمنع التصيد الاحتيالي
  • تمييز الروابط الخارجية: تمييز واضح للروابط الخارجية في الإشعارات

أمان رفع الملفات (File Upload Security)

  • تحقق نوع الملف: Whitelist لأنواع الملفات المسموح بها
  • حد أقصى للحجم: تحديد حد أقصى لحجم الملفات القابلة للرفع
  • فحص تلقائي: فحص تلقائي ضد الملفات الضارة (Virus Scanning)
  • التخزين الآمن: التخزين في مساحة معزولة وآمنة

إدارة المستخدمين وصلاحياتهم

  • كل مستخدم مرتبط بـ user_id داخلي
  • الأدوار: Owner → كامل الصلاحيات، Scholar → كتابة المقالات والفتاوى فقط، User → استخدام المنصة العادي
  • يمكن إضافة أدوار جديدة حسب الحاجة مع تحديد صلاحيات دقيقة
  • واجهة admin.marje3.app تظهر لكل role ما يناسب صلاحياته فقط

بيانات المستخدم

  • لكل مستخدم معلومات أساسية: الاسم، البريد، role، آخر تسجيل دخول، تفضيلات اللغة، الحالة النشطة
  • تخصيص واجهة home.marje3.app حسب تفضيلات كل مستخدم

سجل النشاطات

  • لكل مستخدم سجل تغييرات مهمة (تسجيل دخول، تعديل محتوى، إرسال إشعار)
  • تتبع جميع الأنشطة المهمة لكل مستخدم مرتبطة بـ user_id
  • Audit Logs للأدمن (سجلات التدقيق للمسؤولين): كل إجراء إداري يُسجل ولا يمكن حذفه، Logs محمية من التعديل، تتبع كامل لجميع الإجراءات الإدارية مع timestamp و user_id، سجلات غير قابلة للتغيير أو الحذف لضمان الشفافية والمساءلة

الميزات الرئيسية

المحتوى الديني

  • حفظ الآيات والأحاديث مع الشرح والمصادر
  • مقاطع علمية ودينية وقنوات المشايخ
  • كلمات الأناشيد، القصائد، الأوراد، الأذكار، والتحصينات

التطبيق والمنصات

  • نسخة ويب تعمل كـ PWA (Progressive Web App)
  • تطبيق واحد يغطي جميع المنصات: iOS، Android، Windows، Web

اللغات

  • المحتوى يُدار لكل لغة في قاعدة البيانات مباشرة
  • واجهة للتبديل بين العربية، الإنجليزية، ولغات إضافية
  • دعم عرض المحتوى نفسه بعدة لغات دون أي تعارض

الميزات الدينية

  • مواقيت الصلاة والقبلة حسب الموقع الجغرافي
  • إمكانية تنزيل القرآن والملفات والكتب والفتاوى للعمل بدون إنترنت

العمل بدون إنترنت

  • إمكانية تنزيل كل البيانات المرغوبة على الجهاز: كتب، فتاوى، أذكار
  • استخدام هذه البيانات بدون اتصال بالإنترنت بشكل مباشر وموثوق
  • تحديث المحتوى: آلية لتحديث المحتوى عند العودة للاتصال بالإنترنت
  • حماية حقوق النشر: حماية حقوق النشر للمحتوى المحمل

سهولة الاستخدام

  • دعم قارئ الشاشة للمستخدمين ضعاف البصر
  • تكبير الخط حسب احتياجات المستخدم
  • ألوان مناسبة للمستخدمين ضعاف البصر
  • واجهة قابلة للتخصيص لسهولة الاستخدام
  • تكامل ARIA Attributes: لكل عناصر الواجهة لضمان قراءة صحيحة بواسطة screen readers
  • اختبارات الوصولية: أدوات لفحص WCAG 2.1 على كل صفحة تلقائيًا قبل النشر
  • التنقل عبر لوحة المفاتيح: جميع الأزرار والروابط قابلة للوصول بدون ماوس
  • اختيار تباين الألوان: أكثر من مستوى لتناسب مختلف حالات ضعف البصر

تحسين محركات البحث (SEO)

  • عناوين الصفحات: لكل صفحة عنوان فريد ووصف دقيق يعكس محتواها
  • Meta Tags: وصف وكلمات مفتاحية لكل صفحة
  • Sitemap (XML): ملف XML لجميع الصفحات لتسهيل فهرستها لمحركات البحث - ملاحظة مهمة: يتم إضافة Sitemap بعد أن يكون الموقع يعمل 100% مع نجاح جميع الفحوصات والاختبارات - يجب أن يكون الموقع جاهزًا بالكامل قبل إرسال Sitemap لمحركات البحث (Google، Bing، وجميع محركات البحث الأخرى)
  • robots.txt: ملف robots.txt لتوجيه محركات البحث حول الصفحات المسموح بفهرستها - ملاحظة مهمة: يتم إضافة robots.txt بعد أن يكون الموقع يعمل 100% مع نجاح جميع الفحوصات والاختبارات - يجب أن يكون الموقع جاهزًا بالكامل قبل نشر robots.txt لمحركات البحث (Google، Bing، وجميع محركات البحث الأخرى)
  • Structured Data: استخدام Schema.org لتحديد نوع المحتوى
  • URLs صديقة لمحركات البحث: روابط واضحة، قصيرة، تحتوي على الكلمات المفتاحية
  • تحسين الصور: استخدام ALT Tags ووصف لكل صورة
  • Mobile SEO: التأكد من تجربة مستخدم ممتازة على الجوال

التسويق ووسائل التواصل

  • أزرار المشاركة: لكل محتوى أزرار مشاركة مباشرة على فيسبوك، تويتر، واتساب، لينكدإن، ونسخ الرابط
  • Meta Tags للسوشال: Open Graph وTwitter Cards لكل صفحة
  • تشجيع المشاركة داخل المنصة: إمكانية تقييم المحتوى أو إضافة إلى المفضلة
  • روابط الحملات: روابط مشاركة مخصصة لكل حملة مع تتبع الأداء

شريط الوصول السريع

  • إمكانية إضافة أي رابط من موقع مرجع للوصول السريع
  • خاص بكل مستخدم حسب تفضيلاته
  • سهولة التنقل بين الصفحات المفضلة

البنية التقنية

Technology Stack / التقنيات المستخدمة

Frontend Technologies / تقنيات الواجهة الأمامية

  • Framework: Next.js (React-based framework for production)
  • Language: TypeScript (for type safety and better development experience)
  • UI Library: React (component-based UI library)
  • Styling: CSS Modules / Tailwind CSS (for responsive design)
  • PWA: Next.js PWA support (Progressive Web App capabilities)
  • State Management: React Context / Zustand / Redux (as needed)
  • Internationalization: next-i18next (for Arabic/English support)

Backend Technologies / تقنيات الخادم

  • Runtime: Node.js
  • Framework: Express.js / NestJS (API framework)
  • Language: TypeScript / JavaScript
  • API Architecture: RESTful API (with potential GraphQL for complex queries)
  • Authentication: Google OAuth 2.0, JWT (JSON Web Tokens)

Database & Storage / قاعدة البيانات والتخزين

  • Database: PostgreSQL / MySQL (relational database for structured data)
  • ORM/Query Builder: Prisma / TypeORM / Sequelize
  • File Storage: AWS S3 (for media files, documents, backups)
  • Caching: Redis (for session management and caching)

Cloud Services & Infrastructure / الخدمات السحابية والبنية التحتية

  • Cloud Provider: AWS (Amazon Web Services) - EC2/ECS/Lambda, RDS, S3, CloudFront, Route 53, CloudWatch
  • CDN: Cloudflare - Content delivery network, DDoS protection, SSL/TLS certificates, caching
  • Code Repository: GitHub - Version control (Git), CI/CD pipelines (GitHub Actions), code review and collaboration

DevOps & Deployment / DevOps والنشر

  • CI/CD: GitHub Actions - Automated testing, building, deployment, environment management
  • Monitoring: CloudWatch (AWS), Prometheus/Grafana (for metrics), Sentry (for error tracking)
  • Logging: CloudWatch Logs / ELK Stack

Security & Testing / الأمان والاختبار

  • Security Scanning: Dependency vulnerability scanning (npm audit, Snyk), Code security analysis (SonarQube)
  • Testing Frameworks: Jest (unit testing), React Testing Library (component testing), Playwright/Cypress (E2E testing)
  • Security Tools: OWASP ZAP (penetration testing), SSL Labs (SSL/TLS testing)

Additional Tools & Services / أدوات وخدمات إضافية

  • Email Service: AWS SES / SendGrid (for transactional emails)
  • Push Notifications: Firebase Cloud Messaging / OneSignal
  • Analytics: Google Analytics / Custom analytics dashboard
  • Search: Elasticsearch / Algolia (for advanced search capabilities)
  • AI/ML Services: OpenAI API / Custom AI models (for search and recommendations)

قاعدة البيانات والملفات

  • قاعدة بيانات واحدة وملفات واحدة لكل السب دومينز والصفحات
  • جميع الملفات والبيانات مخزنة على api.marje3.app
  • كل صفحة/سب دومين يسحب فقط البيانات اللي يحتاجها من خلال الـ API
  • كل جدول مربوط بـ user_id
  • Data Residency (إقامة البيانات): تحديد AWS Region أساسي لتخزين البيانات، منع نقل البيانات خارج المنطقة المحددة إلا عند وجود ضرورة تقنية، الامتثال لمتطلبات إقامة البيانات حسب القوانين المحلية
  • Data Classification (تصنيف البيانات): تصنيف البيانات إلى Public / Internal / Sensitive، تطبيق سياسات مختلفة لكل فئة (تخزين، وصول، تشفير)، تحديد مستوى الحماية المناسب لكل فئة بيانات

سب دومينز وربطهم

  • كل سب دومين مستقل من حيث الملفات والواجهة، لكنه مرتبط بالـ API على api.marje3.app
  • المصادقة المشتركة (SSO): SSO موحّد عبر جميع السب دومينز، Token مشترك بين جميع السب دومينز، Cookie policy واضحة (SameSite)، Refresh token mechanism موثق

الأمان

  • ملاحظة مهمة: هذا المشروع لديه أعداء معروفون جيدًا، وإذا أصبح المشروع ناجحًا جدًا، سيحاولون فعل أي شيء لإسقاطه. لذلك يجب تطبيق أعلى معايير الأمان والحماية.
  • حماية من إساءة الاستخدام والهجمات: يتم تطبيق سياسات حماية من إساءة الاستخدام والهجمات (Rate Limiting، WAF، DDoS Protection) لتقليل المخاطر الأمنية ومنع أي استهلاك غير طبيعي قد يؤدي إلى تكاليف غير متوقعة.
  • مبادئ الأمان الأساسية: 1) الحماية قبل كل شيء - أي ميزة أو محتوى جديد لازم يتم اختباره ضد أي هجوم محتمل قبل الإطلاق، 2) الدفاع متعدد الطبقات - Cloudflare/WAF/DDoS Protection على كل مستويات الوصول، Rate Limiting صارم، IP Blacklisting/Geoblocking، Authentication قوية (2FA, Tokens)، Logging & Monitoring مع تنبيهات فورية، 3) خطط الطوارئ - Hard Spending & Traffic Caps، خطط Backup & Recovery سريعة، سيناريوهات Incident Response محددة، 4) التصميم من البداية ضد الهجمات - Feature Flags، Rate-limited Search Index، Minimal Attack Surface، 5) الثقافة الأمنية دائماً حاضرة - كل شخص واعي لمخاطر الهجوم، أي محتوى حساس يتم مراقبته ومؤمن قبل الإطلاق
  • تشفير جميع البيانات الحساسة في قاعدة البيانات
  • حماية الـ API بمفاتيح وصول أو OAuth
  • مراقبة النشاطات غير الطبيعية لتجنب الحسابات الوهمية
  • Rate Limiting: تحديد حد أقصى لعدد الطلبات لكل مستخدم/IP لمنع الإساءة - خوارزمية: Sliding Window - Headers: مفعّلة (X-RateLimit-*) - IPs موثوقة (admin / internal) مستثناة
  • Login Attempts: تحديد عدد محاولات تسجيل الدخول (مثلاً 5 محاولات خلال 10 دقائق) قبل حظر مؤقت
  • API Requests Limits: /search → 50 طلب/دقيقة، /download → 10 طلب/دقيقة، /notifications/send → 5 طلب/دقيقة
  • Download Limits: الحد على حجم أو عدد الملفات القابلة للتحميل خلال فترة محددة
  • Burst Control: التحكم في الزيادات المفاجئة بالطلبات لتجنب هجمات الـ DDoS
  • API Versioning: نظام إصدارات للـ API لضمان التوافق
  • Rate Limiting صارم على: تسجيل الدخول، API، التحميل
  • Cloudflare DDoS Protection: حجب تلقائي للهجمات قبل وصولها للسيرفر
  • WAF Rules (Web Application Firewall): حظر الدول أو الأنماط المشبوهة عند الحاجة
  • Request Validation: منع الطلبات غير المكتملة أو المشبوهة مبكرًا
  • IP Blacklisting / Geoblocking: حظر الدول أو الشبكات المشبوهة
  • Minimal Attack Surface: تقليل النقاط اللي ممكن يستغلها أي مهاجم
  • CORS Policies: سياسات صارمة للـ CORS لحماية المنصة
  • Session Management: إدارة الجلسات مع انتهاء صلاحية تلقائي
  • Key Management (إدارة المفاتيح): إدارة مفاتيح التشفير (KMS)، تدوير المفاتيح دوريًا، فصل مفاتيح الإنتاج عن الاختبار
  • Secrets Management (إدارة الأسرار): عدم تخزين أي secrets داخل الكود، استخدام Secret Manager (AWS Secrets Manager / Environment Variables)، جميع الأسرار الحساسة (API keys, database passwords, etc.) محفوظة في Secret Manager
  • Data Integrity (سلامة البيانات): Hash للمحتوى الحساس، كشف أي تعديل غير مصرح به، التحقق من سلامة البيانات بشكل دوري

اختبارات الأمان

  • Penetration Testing: محاكاة هجمات حقيقية على المنصة لاكتشاف نقاط الضعف
  • Vulnerability Scanning: استخدام أدوات آلية لفحص المنصة بشكل دوري لاكتشاف ثغرات معروفة
  • Security Audits: مراجعة السياسات، الصلاحيات، إعدادات الخوادم، تشفير البيانات

النسخ الاحتياطي

  • خطة يومية/أسبوعية لقاعدة البيانات والملفات المهمة
  • نسخ احتياطي تلقائي لضمان استمرارية الخدمة
  • تشفير النسخ الاحتياطي: جميع النسخ الاحتياطية مشفرة
  • فترة الاحتفاظ: تحديد فترة الاحتفاظ بالنسخ الاحتياطية (مثل 30/90 يوم)
  • اختبار الاستعادة: اختبار دوري لعملية استعادة البيانات

الأداء

  • مؤشرات سرعة التحميل لكل سب دومين
  • تحسين PWA للتحميل السريع
  • CDN للعناصر الثابتة: استخدام CDN للصور والملفات الثابتة
  • فهرسة قاعدة البيانات: فهرسة على user_id والحقول المستخدمة بكثرة

معالجة الأخطاء

  • آلية للإبلاغ عن الأخطاء التقنية تلقائيًا للـ admin
  • تتبع الأخطاء وتسجيلها في سجل منفصل
  • تنظيف معلومات الأخطاء: إزالة المعلومات الحساسة من رسائل الخطأ

المراقبة والتنبيهات في الوقت الحقيقي

  • Alerting في الوقت الحقيقي: إرسال تنبيهات فور وقوع أي خطأ تقني أو نشاط غير طبيعي (email, SMS, Slack)
  • Dashboard مركزي: لوحة تعرض مؤشرات الأداء لكل سب دومين مع الرسوم البيانية للـ CPU, Memory, Load, Response Time
  • سجلات مفصّلة: لكل طلب API، تسجيل الوقت، نوع العملية، المستخدم، نتيجة العملية - خصوصية السجلات: بدون IP خام بعد anonymization - فترة الاحتفاظ: Retention محدد (مثلاً 90 يوم) - لا Logs حساسة: لا تظهر Logs حساسة للمستخدم
  • مستوى التحذير: تصنيف الأخطاء حسب الأهمية (Critical, Warning, Info)
  • مراقبة uptime و latency: قياس الاستجابة لكل endpoint وسب دومين بشكل دوري
  • Time Synchronization (تزامن الوقت): توحيد الوقت (UTC)، مهم للسجلات، الإشعارات، والتحليلات، جميع السجلات والأحداث تستخدم UTC لضمان الدقة والاتساق
  • Cost Monitoring (مراقبة التكاليف): تنبيهات فورية عند ارتفاع تكاليف AWS وCDN، تحديد حدود إنفاق شهرية لتجنّب أي تجاوز غير متوقّع، تتبع التكاليف بشكل دوري وتحليل الاتجاهات، تنبيه عند تجاوز 50% / 80% من الميزانية الشهرية، إيقاف أو تقييد بعض الخدمات تلقائيًا عند الحد الأعلى
  • كيف يزيد الهجوم التكلفة: Bandwidth (طلبات وهمية = نقل بيانات أكثر = فاتورة أعلى)، CDN (كثرة الطلبات على الملفات الثابتة ترفع الاستهلاك)، API / Compute (كل request يستهلك CPU ووقت تشغيل)، Logs & Monitoring (تضخّم السجلات بسبب كثرة الأخطاء)، Storage مؤقت (تراكم بيانات أو كاش غير منضبط)

خطة الطوارئ التقنية

  • لو حصل خلل أو تسريب، طريقة التعامل مع المحتوى الحساس
  • آلية سريعة للتعامل مع الحالات الطارئة التقنية
  • حماية البيانات والمحتوى في حالة الطوارئ
  • سيناريوهات Incident Response محددة: من يحظر، من يراقب، من يتدخل، وغيرها
  • خطط Backup & Recovery سريعة: تضمن عدم خسارة المحتوى أو توقف الخدمة
  • Hard Spending & Traffic Caps: لتجنب استغلال الهجمات للرفع على التكلفة
  • RTO/RPO محددة: تحديد أهداف وقت الاستعادة (RTO) ونقطة الاستعادة (RPO)
  • Failover تلقائي: آليات التحويل التلقائي في حالة فشل الخادم
  • خطة التواصل: خطة واضحة للتواصل مع المستخدمين في حالة الطوارئ
  • Redundancy & Load Balancing: تخطيط للنسخ الاحتياطية وموازنة الأحمال
  • Maintenance Mode (وضع الصيانة): وضع صيانة رسمي، رسالة موحدة للمستخدمين، إمكانية تفعيل وضع الصيانة بسهولة عند الحاجة، إشعار مسبق للمستخدمين عند التخطيط للصيانة

سياسة الخصوصية وشروط الاستخدام

  • فتح الحساب يعني الموافقة التلقائية على الشروط
  • شرح حماية البيانات، حقوق المستخدم، وحقوق المنصة
  • ذكر واضح لسياسة جمع البيانات والـ IP
  • توضيح حقوق نشر المحتوى وحقوق المشايخ والمؤلفين
  • الموافقة الصريحة: موافقة صريحة على جمع البيانات (خاصة للمستخدمين في الاتحاد الأوروبي)
  • حق حذف البيانات (Right to be Forgotten): إجراءات واضحة لحذف بيانات المستخدم عند الطلب - حذف متسلسل (Cascading): حذف جميع البيانات المرتبطة - النسخ الاحتياطية: البيانات في النسخ الاحتياطية تُحذف عند انتهاء دورة الاحتفاظ - مهلة التنفيذ: مثلاً 30 يوم لتنفيذ الطلب - استثناءات قانونية: استثناء فقط للبيانات المطلوبة قانونيًا (إن وُجدت)
  • حق تصدير البيانات (Data Portability): المستخدم يمكنه طلب نسخة من بياناته بصيغة قابلة للتحميل

تحليلات الحملات بشكل متقدم

  • تقارير حسب الوقت والموقع: معرفة أيام الذروة، المناطق الأكثر نشاطًا
  • تحليلات سلوك المستخدم: الصفحات التي وصل لها بعد النقر على رابط الحملة، مدة البقاء، مسار التصفح
  • تصدير البيانات: إمكانية تصدير CSV/Excel للتحاليل والتقارير
  • Dashboard مركزي: عرض كل هذه المؤشرات بشكل رسومي في admin.marje3.app

IP للمسجلين فقط

  • تسجيل الـ IP وربطه بـ user_id عند تسجيل الدخول
  • الغرض: الأمان، منع الحسابات الوهمية، وتحسين الإحصاءات الداخلية
  • ذكر واضح في سياسة الخصوصية
  • الموافقة الصريحة: موافقة صريحة على ربط IP بـ user_id
  • معالجة IP: يُخزن نصًّا لمدة 90 يوم، بعدها يُ anonymize، يُحذف نهائيًا بعد سنة، مذكور صراحة في سياسة الخصوصية

مراجعة المحتوى

  • نظام لتتبع مراحل مراجعة المقالات، الفتاوى، الكتب قبل النشر
  • حالات المحتوى: مسودة، قيد المراجعة، معتمد، منشور
  • آلية الموافقة النهائية للفتاوى: تحديد خطوات واضحة لمنع أي نشر قبل أبروفال من الأدمن بعد مراجعة المشايخ
  • تحديد المسؤوليات: لكل مرحلة مراجعة، من كتابة → مراجعة → تعديل → اعتماد، يكون واضح من المسؤول عن كل خطوة
  • سجل التعديلات على الفتاوى: حفظ نسخة من كل تعديل حتى لو تم رفضه أو تعديل لاحق
  • سياسة النشر والحقوق: توضيح حقوق نشر المحتوى وحقوق المشايخ والمؤلفين
  • تجربة المستخدم النهائي: المستخدم يشاهد المحتوى المعتمد فقط بدون أي محتوى قيد المراجعة
  • حل الخلافات بين المراجعين: آلية لحل الخلافات عندما يختلف مراجع مع آخر
  • نظام الإصدارات: نظام إصدارات كامل للمحتوى مع إمكانية الرجوع للإصدارات السابقة
  • Legal Disclaimer للفتاوى (إخلاء مسؤولية قانوني للفتاوى): الفتاوى منشورة من مفتين معتمدين ومراجَعة من مشايخ مختصين، الفتوى قد تختلف باختلاف الزمان والمكان والحال، المنصة مسؤولة عن صحة النقل والمراجعة، لا عن التطبيق الفردي الخاطئ، إخلاء مسؤولية واضح في كل فتوى منشورة
  • Abuse Reporting (الإبلاغ عن الإساءة): زر إبلاغ عن محتوى أو إساءة، Workflow واضح للتعامل مع البلاغات، نظام لتتبع ومعالجة البلاغات بشكل منظم، إشعارات للمسؤولين عند استلام بلاغ جديد، آلية لمراجعة البلاغات واتخاذ الإجراءات المناسبة

البنية التحتية والاستضافة (Infrastructure & Hosting)

  • استضافة سحابية بالكامل: يعتمد مشروع مرجع على استضافة سحابية بالكامل بدون أي خوادم محلية - جميع البيئات (الموقع، الـ API، السب دومينز) تعمل على بنية سحابية موثوقة وقابلة للتوسع - لا يتم تشغيل أي جزء من النظام من أجهزة شخصية أو شبكات منزلية
  • إدارة الكود والنشر: الكود المصدري محفوظ في مستودع مركزي واحد (GitHub) - المستودع هو المصدر الوحيد للحقيقة (Single Source of Truth) - أي تحديث على الكود يتم نشره تلقائيًا على المنصة بدون تدخل يدوي مباشر بالخوادم
  • الأداء وتوزيع المحتوى (CDN): استخدام شبكة توزيع محتوى (Cloudflare CDN) لتسريع تحميل الصفحات والملفات - يتم تقديم المحتوى للمستخدم من أقرب نقطة جغرافية له - CDN تقلل الضغط على الخوادم الأساسية وتحسّن الأداء في جميع الدول - CDN مسؤولة عن: تسريع الصور والملفات الثابتة، التخزين المؤقت (Caching)، الحماية الأساسية من الهجمات الشائعة
  • الاعتمادية والاستمرارية: النظام مصمم بدون نقطة فشل واحدة (No Single Point of Failure) - في حال تعطل جزء من الخدمة، لا يتأثر المستخدم النهائي بشكل مباشر - البنية تدعم التوسع الأفقي حسب عدد المستخدمين والزيارات
  • الأمان: لا يوجد وصول مباشر للخوادم من المستخدمين أو المطورين - جميع الطلبات تمر عبر طبقات حماية وشبكة CDN - البنية تقلل المخاطر المرتبطة بسرعة الإنترنت أو الموقع الجغرافي للمشغّل
  • مبدأ التصميم: سهولة التشغيل (التركيز على تطوير المحتوى والميزات، لا إدارة الخوادم) - الاحترافية (نفس بنية المشاريع العالمية) - الاستدامة (قابلية النمو بدون إعادة بناء جذرية مستقبلًا)

الهدف

مرجع متكامل يجمع بين: العلم (المحتوى العلمي الموثوق)، التطبيق (أدوات عملية للعبادة اليومية)، الدعوة (نشر الخير والمعرفة)، والمشاركة المجتمعية (الحملات والمبادرات). مع واجهة بسيطة وسهلة الاستخدام لجميع الأعمار.

التصميم والهيكلية

  • هيكلية الهيد والفوتر: تصميم احترافي ومتسق عبر جميع الصفحات
  • ترتيب المحتوى: تنظيم واضح ومنطقي للمحتوى
  • تجربة المستخدم: واجهة سهلة الاستخدام وسريعة الاستجابة
  • سهولة التنقل: تنقل سلس بين الصفحات والأقسام
  • التصميم الاحترافي: التأكيد على جودة التصميم والهيكلية في جميع أنحاء المنصة

CI/CD والنشر

  • CI/CD: كل تحديث للكود يمر عبر Pipeline آلي: Build → Test → Lint → Deploy
  • اختبارات تلقائية: Unit + Integration + End-to-End قبل الدمج
  • إمكانية Rollback: العودة لإصدار سابق إذا حصل خطأ بعد النشر
  • فصل بيئة Staging عن Production لاختبار التحديثات قبل النشر النهائي
  • نشر تدريجي (Canary Deployment) لتقليل المخاطر
  • إدارة الإصدارات: Versioning لكل تحديث لتسهيل الرجوع عند الحاجة
  • Feature Flags (أعلام الميزات): إمكانية تفعيل أو تعطيل أي ميزة دون الحاجة لنشر تحديث جديد، تطبيق الإطلاق التدريجي (Rollout) للمميزات الحساسة، لتفعيل/تعطيل أي ميزة عند ظهور أي هجوم، إدارة الميزات الجديدة بشكل آمن ومرن، إمكانية إرجاع الميزات بسرعة في حالة المشاكل
